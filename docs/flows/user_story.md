# 滑雪課程預約系統 — 使用者故事 (User Stories)

---

## 1. 核心使用者流程 (Core User Flows)

### 1.1 作為學員 (Learner)

- **登入與註冊：** 我希望能透過 **Google 或 LINE 一鍵登入**，這樣我就不需要記住複雜的密碼，並能快速進入預約流程。
- **資料同步：** 當我第一次使用社交登入時，我希望系統能**自動建立我的帳戶**，並讓我填寫滑雪程度與語言偏好，以便獲得個性化的推薦。
- **尋找課程：** 我希望能根據**日期、地區或教練**來篩選課程，並能即時看到剩餘名額，避免選到已經額滿的時段。
- **預約佔位：** 當我點擊預約進入支付頁面時，我希望系統能為我**保留名額 10 分鐘**，讓我有足夠時間完成付款，而不必擔心名額被搶走。
- **多元支付：** 我希望支援信用卡、Apple Pay 與 LINE Pay，讓支付流程更加便捷。
- **訂單管理：** 我希望能隨時在我的儀表板看到即將到來的課程，並能下載**電子發票**與查看退款政策。

### 1.2 作為教練 (Coach)

- **排程管理：** 我希望能設定我每週固定開放預約的時段，並能針對特定日期進行調整（如請假或休息）。
- **學生管理：** 我希望能看到已經預約我課程的學生名單及其滑雪程度，以便我能提前準備教學內容。
- **分潤追蹤：** 我希望能定期看到我的**分潤報表**（目前為 70%），並能導出 CSV 檔案進行對帳。

### 1.3 作為管理者 (Admin)

- **多語系校正：** 我希望能直接在後台介面修改課程名稱與描述，並能**立即生效**，而不需要等待工程師重新發布程式。
- **系統監控：** 我希望能查看所有訂單的狀態與金流回傳紀錄，並能處理特殊的退款與發票作廢請求。

---

## 2. 邊界情況 (Edge Cases)

- **超賣防護：** 當兩個人同時搶最後一個名額時，系統應確保只有其中一人成功，另一人應收到「名額已額滿」的錯誤提示。
- **支付超時：** 如果學員未在 10 分鐘內付款，系統應自動釋放名額，並將訂單標記為「已過期」。
- **退改規則：** 如果我在開課前 48 小時內取消預約，系統應明確告知我不予退款，並阻止我發起退款請求。
